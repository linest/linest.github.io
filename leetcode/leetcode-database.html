Leetcode(数据库篇)

LeetCode竟然开数据库的新坑了，填之~
SQL相对比较简单，就都集中写在本文吧

归纳起来比较重要的点就是如何处理连续记录，如何获取排序索引

函数

  • ROUND(x, d): 数字x保留d位小数
  • DATEDIFF(d1,d2): d1与d2日期差值
  • IFNULL(exp, v): exp为空时取v值
  • IF(exp, a, b): exp成立时取a,否则取b

175. Combine Two Tables

人员表和地址表，求人员对应的地址情况，即使没地址也列出

以人员为基准，直接left join

176. Second Highest Salary

求第二高薪水，如果没有返回null

注意点

  • 薪水可能重复，要去重。可考虑手动实现去重的distinct或者有自动去重效果的max
  • 没值时不能返回空，要返回null

Mysql自带limit特性可以方便求得nth大值
LIMIT offset,count 从第offset(0-base)条开始取，取count条

但是没值时返回空，不是返回null，需要再处理，利用IFNULL函数

更通行的解法是用子查询，第二大的值是比最大值小的记录中的最大值，动用了函数，没值时会返回null

177. Nth Highest

求第n高的薪水，重复的算一个，如果没有返回null

依然可以用limit的特性，用自定义函数没有会返回null
SET N = N -1直接自赋值可以省去再单独定义一个变量

178. Rank Scores

分值表包含id和分数，求每个分数对应排名的榜单，同分同名次，名次连续不跳跃

笛卡尔积可以实现双重循环的效果,对每一条记录用比较可以滤出比它大的值，分组统计数量即为排名
这个方法很有用，可以用来求Top N

180. Consecutive Numbers

求至少连续3个的数字
供操作的表给出了Id并且是连续的，这个很重要，相当于给出了数字的索引
数据库跨行的逻辑很难，同一行的逻辑比较简单，既然是连续三个，直观的方法是将三个值弄到同一行去
可以根据索引进行两次阶梯式Join,结果就是每一行包含一条记录以及之前的两条记录
数据可能包含超过三个连续的情况，因而取Distinct即可
Join是非常重量级的操作，如果求连续多个(大于3)的话会比较难，也许这就是题目给出3的隐含意味？

181. Employees Earning More Than Their Managers

员工表记录了自己的工资和上级的Id, 求比上级挣得多的员工

自身JOIN,直接把自己和上级弄到一行然后比较即可

182. Duplicate Emails

人员表包含id和邮箱，检测重复的邮箱记录

根据邮箱分个组计数即可

183. Customers Who Never Order

给出了客户表和订单表, 求没下单的客户
客户表为基准left Join一下找出null即可

184. Department Highest Salary

员工表包含id,名字，薪水和部门id, 求各部门最高工资的员工

问题分三步

  1. 先用一个子查询分组求出各部门最高工资
  2. 在各部门中找出拿到最高工资的员工
  3. 最后翻译出部门的名字

题目没要求最终输出时各部门间的顺序，没必要排序

185. Department Top Three Salaries

求各部门工资前3的员工
和上一题类似分三步，第一步变成了求每个部门的Top3工资
利用178. Rank Scores的思路，先用子查询可以求得单员工在部门的排名情况得出排名表Chart，在此基础上求出部门的Top3工资表TopSalary
题目没要求最终输出时各部门间顺序及部门内各员工顺序，最后不用排序

196. Delete Duplicate Emails

删掉重复的邮箱记录，保留Id最小的

先分组求出要保留的Id,再将之外的记录全删掉
使用not in时又包了一层，这是因为MySQL的特性，官方文档说

You cannot update a table and select from the same table in a subquery.

197. Rising Temperature

天气表包含Id,日期和温度，求比前一天温度高的记录

相邻记录的逻辑，把每天和前一天Join在一起

262. Trips and Users

计算没被禁的顾客的订单取消率
有订单表包含顾客司机Id,状态(完成，顾客取消，司机取消)和时间
人员表用角色区分（顾客，司机，合伙人），还有标识是否被禁了
这是场景就是Uber嘛。。。

这题考察条件统计，sum+if或case的应用
SUM(IF(Status != 'completed',1,0))可以简写成SUM(Status != 'completed')

511. Game Play Analysis I

活动表包含玩家id,设备id,日期，游戏局数，玩家id和日期是主键，统计玩家首次登陆日期

512. Game Play Analysis II

求出首次登陆id和日期后，再次关联出设备

534. Game Play Analysis III

活动表包含玩家id,设备id,日期，游戏局数，玩家id和日期是主键，统计玩家每个日期累计玩了多少游戏

前序和问题，首先自身关联，求出每个日期关联前面日期，再分组统计求和

550. Game Play Analysis IV

活动表包含玩家id,设备id,日期，游戏局数，玩家id和日期是主键，统计首日登陆后连续登陆用户占总用户数的百分比

统计次日登陆占总登陆数
