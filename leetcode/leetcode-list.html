LeetCode-链表篇

总结

核心：找中点，反转，找前驱节点，合并

关注点

  • null检查，只要访问指针，就应确保当前节点不是null
  • 有环
  • 有序
  • 变更目标(头节点，尾节点，一般节点)

技巧

  • 伪头节点，避免头结点发生变化
  • 前后指针定位

插入和删除

关键是找到前驱节点

❀19. Remove Nth Node From End of List

删除倒数第n节点，保证n合法

首先前后指针定位要删的节点，然后考虑被删节点角色，如果头节点直接删，如果中间或者尾节点找前驱删

❀237. Delete Node in a Linked List

只给要删除的节点，实现删除效果

没有给头结点，因此无法通过正常手段找到前一节点做删除，采用节点复用的方式，后面一个节点的前移，转变成删除后一节点

❀708 Insert into a Cyclic Sorted List

有序循环链表插入

考虑情况，插入最小，插入最大，插入中间

编织新链表

❀203. Remove Linked List Elements

删除链表中的特定值

因为头也可能删除，因此想象成构建新链表，利用伪头直接接续
最后节点要手动断开，避免和原节点相连

❀83. Remove Duplicates from Sorted List

有序链表去重

用不重复的值构建新链表

❀82. Remove Duplicates from Sorted List II

有序链表只保留非重复

判断当前和后面是否相等，如果相等，循环跳过这个元素

328. Odd Even Linked List

奇数位置节点在前，偶数位置在后

开两个链表分别收集，最后合起来

86. Partition List

以一个数值为基点，划分链表，保持相对顺序，空间O(1)

开两个列表分别收集，再合并

❀206. Reverse Linked List

原地反转链表

  • 迭代
    三指针推进，前后当前
  • 递归
  • 尾递归

❀24. Swap Nodes in Pairs

两两一组反转，k组反转的简化版

保持前指针，两节点指针，后指针，逐步交换即可

❀92. Reverse Linked List II

翻转m到n部分，保证mn合法

思路是先定位m和n节点并找到前后节点，子链表脱离，反转子链表，再接回去
注意可能头节点会变更

❀25. Reverse Nodes in k-Group

k个节点为一组，组内翻转

每k个节点脱离，翻转，再接回

节点定位

❀876. Middle of the Linked List

找到链表中点

奇数节点时中间，偶数节点时偏右

奇数节点时中间，偶数节点时偏左

❀234. Palindrome Linked List

判断是否回文，要求空间O(1)

先找到中点，反转右半部分，和左半部分同步遍历比较

  • 如果奇数个节点1->2->3<-4<-5，指向中间翻转，这样左右两部分长度相等
  • 如果偶数个节点1->2->3->4<-5<-6，指向右侧翻转，这样右侧比较短，以右侧为基准即可

143. Reorder List

前半部分和后半部分的逆序交叉

先找中点，后半部分反转，断开成为两部分，两部分合并
中点选择偏左策略，这样便于偶数节点时均衡断开，断开后前部分长度大于等于后半部分

链表相交

❀141. Linked List Cycle

判断链表是否有环

快慢指针，如果有环就会相遇，如果到头了说明没环
原理：两个指针都进环后，快指针每次追赶1步，总会追上

❀142. Linked List Cycle II

如果有环找到环起点

判断成环后，快指针重置到头节点，两指针同步遍历，下次相遇就是环节点
原理：设起点到成环点距离x,环点到相遇点距离y，n步时第一次相遇
n = x + y + 若干圈;2n = x + y + 若干圈;求得n=若干圈，即慢指针总的移动距离是环长的若干倍, 此时再走x,刚好x+若干圈，两者相等相遇

❀160. Intersection of Two Linked Lists

求两个无环链表公共节点，要求空间O(1)

求长度，补齐差值后同步遍历

扩展

  • 如果空间O(N),那么可以采用哈希表，计数找被指向两次的节点
  • 如果节点值有特点，那么可以标记，比如正数变负数
  • 如果节点有序，那么可以比大小对齐
  • 什么特点都没有，那么只能求出长度后进行对齐

如果链表可能有环，需要额外判断

  • 有环和无环不可能相交，要么都无环，要么都有环
  • 无环相交
    • 最后节点不同则不相交
    • Y字型，利用长度差对齐求交点
  • 有环相交
    • 如果环节点相同，说明前期就相交，可以认为环节点为终点
    • 如果环节点不同，那么从一个环节点出发一周内一定能遇到另一个环节点，否则不相交

排序

148. Sort List

链表O(nlogn)排序

归并排序，由于是链表不需要分配额外空间，效率更好
由于链表不能随机访问快速排序不太适合
融合了链表合并，求中点，链表编织操作

147. Insertion Sort List

链表插入排序

常规插入是每次选一个尝试向前插，链表不适合，改为从头尝试向后插
注意头部变化

切分

725. Split Linked List in Parts

尽可能均匀切分链表，前面的可以略长

辅助函数，切分头部指定个数

❀61. Rotate List

循环右移k位

总体思路是按长度切分成两部分，再交换拼接
需要考虑实际移位数，如果总长大于k，那么移动k；如果总长等于k，那么移动0；如果总长小于k，那么移动k%len

复制

❀138. Copy List with Random Pointer

拷贝带有随机指针的链表

难度在于，拷贝过程中，随机指针可能指向后面节点，而后面节点尚未拷贝产生
如果允许空间O(n)，可以先不管随机指针，保存拷贝节点和原节点的对应关系，完成后再遍历一次调整随机指针
如果只允许空间O(1)，可以先把拷贝节点串在原节点后面，这样变相保持来了对应关系，全完成后再把拷贝节点单独剥离
