LeeCode-枚举回溯篇

总结

通常通过DFS和BFS处理

复杂度

  • n个元素所有排列的个数为n!
  • n个元素所有子集个数为2^n-1

关注点

  • 重复
  • 重用
  • 有序
  • 定长/变长

排列

❀46. Permutations

给定一组不重复的数字，生成全部排列

想象成顺次挑选下一个数字，即每个位置关联一个可用集合，有集合大小种选择
递归，每一层尝试所有前面没出现过的数字
复杂度O(n!)

迭代，每次对于前面所有可能，把新数字插入不同位置

❀47. Permutations II

给定一组可重复数字，生成全部排列

递归，由于有重复，无法简单利用数值判定是否前面已经用了，每个数字需要记录是否使用过
还需解决重复排列问题，先将数组排序，保证相同数字在一起。如果当前数字和前面相同，并且前面的未使用，那么当前数字也不用继续。比如11，颠倒是一样的无法区分，因此对相同数字引入次序，必须前面的使用过了，才能使用后面的

迭代，依然是各位置插入，由于有重复，当插入位置和原来位置数字相同，那么停止后续，避免生成重复排列

❀949. Largest Time for Given Digits

给定4个数字，求能构成的最大时间，不能构成返回空串

四个位置，每个位置先不考虑重复使用理论上4种选择
数只有四个，直接开循环，没必要递归，三层循环即可，经过三层只剩下一个会被唯一确定
判定范围直接使用字符串比较，没必要转数字
实质上是求4个可重复数字的所有排列，因为重复计算不影响后果，因此不特殊处理重复

401. Binary Watch

时间数字用二进制表示，给定1的个数求所有时间组合

两个位置，第一个位置12种选择，第二个位置60种选择
直接双重循环遍历所有时间组合，筛选1个数

❀784. Letter Case Permutation

数字字母混合字符串，输出所有字母大小写可能

每个位置两种选择，可以是大写也可以是小写
BFS迭代，初始把原始串进队列，针对字母每一位，把之前结果处理为大写小写两个版本进队列

DFS递归，不断进行大小写分支

17. Letter Combinations of a Phone Number

每个2-9数字映射不同的一组字母，给定数字串，生成所有可能的字母解析

每个数字位置，关联可选字母集合
队列BFS生成

❀22. Generate Parentheses

n对括号，求所有合法序列

每个位置两种选择，可以是左括号，也可是右括号
附加条件合法序列任意前缀左括号大于等于右括号
字符串不可变，不需要恢复现场

DP卡特兰数也可以求解

❀93. Restore IP Addresses

给定字符串，求能构成的所有ip地址

每段可以有1到3个数位构成0到255
总限制条件：划分四段，255以内，不能0开头

组合

❀77. Combinations

1~n数字中k个数字的组合

本身不重复，保持升序，每加入一个元素，后续必须更大

❀78. Subsets

给定一组不重复数字，求全部组合

和排列不同

  • 组合不定长，把过程也作为结果
  • 不同顺序不会产生新结果，数字本身不重复，因此每层都向后尝试即可

迭代, 每次有两个选择，选这个数或者不选，即每一层都会在上一层基础上变为2倍数量

❀90. Subsets II

给定一组可重复数字，求全部组合

递归，先排序，同一层同一数字只处理一个即可，比如111，结果可能1,11,111，分别由三层产生，即结果中第一个1只能由第一个数字1提供

迭代，每层记录新增加的部分，对于重复元素，只在增加的基础上增加
比如111，第一轮可以有0个1,1个1，第二轮只增加出2个1，第三轮只增加出3个1

❀39. Combination Sum

一组不重复正数，同一数字可以重复利用，求和等于指定值的组合

可以重复使用，数字数量无限，因此每层不会直接跳过这个数

迭代，同一值尽管加，直到超过目标

❀40. Combination Sum II

一组可重复数字，数字不可重复利用，求和等于指定值的组合

求重复数字所有组合，验证和值即可

❀216. Combination Sum III

给1~9， 求k个数和为n的组合

求k个元素组合，验证和值
数字不重复，也不能重用，生成排列，同时考虑数字个数和目标和

❀15. 3Sum

一组可重复数字，找出所有3个数和为0的不重复组合

直接按照求可重复数字固定数量组合的常规方式处理，报超时

考虑到一共只有3个数字，选中一个数字后，向后找两个数字和，转化为2sum问题
因为已经有序，2sum可采用双指针法无需建立哈希结构，注意一旦找到，还需要跳过后续重复
总时间复杂度O(n2)

293. Flip Game

只包含±的字符串，操作一次把连续两个+变成两个-，求所有可能

相当于多个++位置，取一个的组合
回溯，遇见两个+就变，复制加入结果后复原，因为只操作一次无需递归

❀294. Flip Game II

交替操作，使对手无法操作就会胜利，求先手能否必胜

遍历每一种情况，递归判定

计算过程中会重复计算，可以建立哈希表缓存

❀131. Palindrome Partitioning

分割一个字符串，使得每部分都是回文，求所有分割方法

字符串分割，实际上是个集合问题，插板法，从全部索引中选出若干个作为分割点，判断每部分是否是回文



一分也是爱~
