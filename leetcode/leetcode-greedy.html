LeetCode-贪心策略篇

策略类型

  • 当前利益最大化，单步积累造就整体最优
  • 后续空间最大化，为后续操作提供便利

❀402. Remove K Digits

从非负数中删除K个数位，保证数字长度大于K，使得整体数字尽量小

整体数字小，尽量让前面的数位小
每遇到一个数位，如果有删除额度，并且比前面数位小，就把前面删除
在上述策略下K可能用不完，比如12345没有数位被删除，最终只保留所需位数即可
前面数字可能动态变化，采用栈实现更容易

561. Array Partition I

长为2n的数组，两两一对从中留下较小的，求留下最大和

两两一对，肯定舍弃一半。剩余数字中最大的肯定留不下，那么不如去保留次大的。依次类推，留下的是位于偶数索引的数字

❀942. DI String Match

D代表下降，I代表上升，给出一个DI构成的升降序列，求一个任意符合升降关系的0~n排列

只顾及相邻元素间的升降关系，0~n数字天然有序，可以记录剩余可用区间最小和最大值

  • 遇到升，优先选择最小的，这样后面确保可以升
  • 遇到降，优先选择最大的，这样后面确保可以降

❀860. Lemonade Change

一个商品5块钱，客户会付5,10,20，初始没钱，排队购买，判定是否能找开

找钱优先找大额，以免后面找不开

575. Distribute Candies

偶数2n根蜡烛，属于多个种类，均分给兄妹，求妹妹最多能获得多少不同种类蜡烛

只要种类超过n种，那么妹妹保证手里一样一个，否则有重复取种类数即可

984. String Without AAA or BBB

构造一个A个a,B个b组成的字符串，使得里面没有三连

没有三连，那么最多连续出现两个
哪个多就尝试多消耗，比如a多就构造aab形式，相等就进行abab交错

❀55. Jump Game

数组每个元素代表在当前位置可向后跳的范围，初始在头部，判断是否能到达尾部

动态规划，从尾部开始计算，判断每个格子所有可能是否可以到达尾部

假设从头可以到达尾部，如果中间某一点可以到达尾部，那么从头也必然可到达中间点,即转化为到达中间点问题
从末尾开始判断，判断自身能不能到达下一目标，目标不断前移，这样不必遍历每个点所有可能范围，每个点只判断一次，O(n)

❀45. Jump Game II

数组每个元素代表在当前位置可向后跳的范围，初始在头部，假设一定能到达尾部，求最少步数

每一步计算能到达最远距离，新的范围作为下一步起跳范围，范围覆盖问题
初始是0，比如能跳到3，那么13范围内都可以一步到达，作为第二步起跳范围，比如第二步最多可达到6，那么46就是两步到达范围

❀754. Reach a Number

初始在0，第n步走距离n，可以向左也可以向右，求到达目标的最少步数

目标在左右其实是对称的，因此先求绝对值，先固定在右边
一直向右走，直到超过目标，如果调整其中一步的方向，正向取消，反向增加，总体上减少2倍距离

  • 如果超过目标偶数距离，那么直接改变其中一步方向即可
  • 如果超过目标奇数距离，那么继续走，因为奇偶交替，会在两步内变成偶数

1221. Split a String in Balanced Strings

等量LR字符构成的字符串，求最多能划分成多少个等量部分

原始就是等量，如果能划分出一段那么剩下的也满足等量，因此只要满足就立刻划分即可

❀392. Is Subsequence

判断s是否是t的子序列

t中依次找s中的字母，找到第一个符合的就可以找下一个，剩下的长度越长能找到的机会越大

如果t很长，反复判断不同的序列s，每次都要遍历一遍效率比较差
收集t中每种字母出现的下标，每种字母都对应一组升序下标，便于t每次二分查找s字母位置

❀521. Longest Uncommon Subsequence I

找最长的非公共子序列，即一个字符串的子序列不是另一字符串的子序列

如果两串相等，那没有，如果不相等那最长的串整串必然不是另一串的子序列
