LeetCode--哈希统计篇

总结

关注点

  • 范围

技巧

  • 范围有限可以考虑使用数组
    • 26个小写字母映射到0~25，int index = c - 'a';
    • 52个大小写一起映射0~51，前半部分小写后半部分大写，A(65)a(97)int index = (c >= 'a') ? c - 'a' : c - 'A' + 26;
  • map.getOrDefault和map.putIfAbsent简化判断

编码映射

❀205. Isomorphic Strings

判断字符串是否同构，即字符可以一一映射到另一字符

一一映射，利用map保存映射，同时用set记录是否用过，避免映射到同一字符

❀290. Word Pattern

判断空格分割出的单词是否符合模式，比如abba模式cc dd dd cc

注意是一一映射，a映射cc后a不能映射别的，同时cc也不能映射别的

804. Unique Morse Code Words

给出字母表的摩斯码，求单词中有几个不同的摩斯码

把每个单词转换摩斯码，去重即可

500. Keyboard Row

找出由键盘同一行字母构成的单词

首先建立字母和行的映射，每个单词判断是否同行

计数

387. First Unique Character in a String

找出第一个只出现一次的字符

第一次遍历统计，第二次遍历查找

884. Uncommon Words from Two Sentences

求两个句子中只出现一次并且在另一句中没有的单词

常规方法是分别求两个句子中只出现一次的单词，再比较
实际上可以转化为求全局只出现一次的单词

存在性

771. Jewels and Stones

统计S串内属于J串字符的个数

建立J串字母表，S串判断

统计计算

1189. Maximum Number of Balloons

判断一组字母可以拼出Balloon单词几次

统计可用字母次数，计算单个字母需求，总次数受最小次数制约

409. Longest Palindrome

判断一组字符可以拼出最长的回文

计数，按照2的倍数累计，如果还剩下字符，可以放中间一个

❀532. K-diff Pairs in an Array

差绝对值为k的数对个数，数对不重复计数，不考虑数对内顺序

先统计每个数字出现次数

  • 如果绝对值差为0，需要出现至少两次
  • 数对去重，单向计算差值

447. Number of Boomerangs

一堆点，判断构成等腰三角的点三元组，区分顺序

每个点尝试作为顶点，计算和其他所有点的距离，相同距离计数
每种距离计算从中取2个的排列数，构成三元组

统计合并

以一个统计为基础，合并另一个
如果只针对两个对象，无需维护两份统计，可以增减计数

❀242. Valid Anagram

判断两个字符串是否相同字符构成

注意如果长度不等，直接否定，否则会误判
第一个字符串建map增加计数，第二个字符串减少计数

389. Find the Difference

两个字符串，后者是前者乱序后随机加一个字符，求加的字符

可以用哈希计数，更加单的方式是根据异或性质，两串汇总只有一个字符是奇数次

383. Ransom Note

判断单词是否可以从杂志中剪出字母拼出

增减法，字母有个数限制用完会减少

❀1002. Find Common Characters

求多个字符串公共字符，多次要重复输出

合并多个统计，求每个字符最小次数
