REST vs SOAP vs RPC

  • Concept
  • Web Service
Web

最近遇到一个系统，它提供了多种版本的API。到底这些版本间有何异同呢，正好借此机会总结下相关知识。

参考：微软的文章

首先名字上就能看出些许差别
RPC:Remote Procedure Call
本地代码跟本地的代理交互，代理和远端交互，代理模仿远端的接口，本地感觉像是直接调用了远端接口一样
REST:Representational State Transfer
一般我们都说RESTful. ful是英语形容词的词尾，RESTful应该理解成一种风格(Style)，感觉是说了看上去像什么样
SOAP:Simple Object Access Protocol
SOAP作为一种数据交换协议(Protocol)，严格定义了协议内容,具体的实现细节

总之 RPC和REST只是描述了一种方法，而SOAP是一种协议，SOAP的部分实现可以采用RPC和RESTful的方案

REST优点

  • RPC由于要和代理交互，变相依赖了远端代码，而REST无依赖，完全解耦合
  • REST用到了HTTP的GET方法，可以充分利用缓存
  • REST是无状态的，每次请求是独立的互不影响
  • REST可以发送多种形式数据，SOAP一般只能XML

SOAP优点

  • REST本身可以是安全的，比如可以应用TLS建立安全的HTTP传输，但是SOAP有成熟的端到端安全方案，支持更灵活复杂的场景
  • RESTful虽然可以用各种协议实现，但是默认实现是基于HTTP协议，如果涉及依赖多协议的场景，SOAP可能会更加适合一点
  • RESTful突出无状态性，相比而言SOAP对事务的操作支持比较好,对事务有一套完整的支持
  • 在WSDL支持下，可以为所有基于SOAP的服务生成代理，客户端编程变得容易
