spring-boot

配置：无xml, 自动装配
集成：starter
部署：jar包，内嵌tomcat

简化开发流程，自动化配置
xml -> 2.5 注解扫描 -> 3.0 config类配置 -> boot打包配置

简化

  • 独立Maven依赖 ——> starter套装
    • 经验总结，包含常见功能
    • 经过测试，确保版本适配
  • Spring配置文件 ——> 注解扫描 + Configuration配置
  • 组件手动装配 ——> 自动感知装配
    • 依赖间整合
  • web.xml配置war包外部Tomcat ——> 内建Tomcat，jar包main运行, 由spring‐boot‐maven‐plugin

Spring Boot CLI
自动检测引入依赖，像脚本一样运行

Actuator
运行情况检测工具

建立

Spring Initializr
https://start.spring.io

通过CLI命令初始化
spring init -dweb,data-jpa,h2,thymeleaf --build gradle example

启动类

@SpringBootApplication
开启组件扫描和自动配置
集合下设功能
@SpringBootConfiguration
@EnableAutoConfiguration
@ComponentScan

main方法，支持jar方式运行，内嵌数据库，内嵌服务器，直接可运行
自动配置maven构建插件，mvn spring-boot:run目标

main方法中SpringApplication.run方法启动应用，启动之前可以进行参数配置，比如寻找可用端口

spring-boot-autoconfigure包含各类配置，4.0条件化配置@Conditional
应用启动时，根据classpath内容自动生效相关配置

spring-boot提供常用基础，允许用户自定义调整

  • 自定义config类
  • application.yml或者application.properties
    • 包外config目录
    • 包外根目录
    • 包内config目录
    • 包内根目录
  • 命令行参数
  • 环境变量

调整依赖
Spring Boot默认使用Logback日志，如果需要使用Log4j，先排除再引入

ErrorViewResolver

Actuator

引入依赖，开启监控端点
REST方式，引入spring-boot-starter-actuator

  • 配置生效情况
    • /beanBean装配情况，类型、依赖等
    • /autoconfig自动配置情况，各条件是否成立
    • /configprops配置属性注入情况，包含默认值
    • /env各种配置
    • /mappings端点映射情况
  • 服务运行状况
    • /metrics各种运行信息，内存，堆，线程池等等
    • /trace内存维护请求信息，包含请求类型，请求响应头等
    • /dump线程快照
    • /health健康状况，磁盘空间，数据库等
  • 操作
    • /shutdown关闭服务
  • 其他
    • /info应用信息，邮箱联系方式等

远程shell方式，引入spring-boot-starter-remote-shell
JMX方式，使用JConsole访问MBean

配置开启所有端点

http://localhost:11111/actuator/beans

部署

脱离内置服务器，比如使用War部署服务器

  1. 修改POM打包<packaging>war</packaging>
  2. 创建SpringBootServletInitializer子类，用于不走main方法入口后，对Servlet进行相关配置
  3. 如果main方法还在，即使war包也可以直接运行
  4. 如果需要彻底去掉内置tomcat,把依赖改成provided

脱离内置数据库

  • 使用Profile显示定义DataSource的bean，但代码中写死
  • 使用spring.datasource前缀的配置项，代码外配置数据源
    激活生产环境配置export SPRING_PROFILES_ACTIVE=production

数据迁移

内置支持

  • Flyway
  • Liquibase
    启动时扫描执行，单独维护表记录schema版本，进而增量执行

开发者工具

添加依赖spring-boot-devtools

热部署

  • 自动重启更新配置和类：
    • spring.devtools.restart.exclude可以自定义不重启的目录，默认下排除/META-INF/resources、/resources、/static、/public和 /templates
    • spring.devtools.restart.enabled可以关闭重启功能
    • spring.devtools.restart.trigger-file可以控制重启时机，修改某个文件后才触发重启
  • 不重启更新静态资源：Run->Edit Configurations->On frame deactivation: Update resources
    • 浏览器安装LiveReload插件可以自动刷新

远程开发

服务设置远程安全码spring.devtools.remote.secret
运行org.springframework.boot.devtools.RemoteSpringApplication，参数是远程部署地址

  • 本地修改自动推送
  • 远程调试，服务添加JVM参数JAVA_OPTS: “-Xdebug -Xrunjdwp:server=y,transport=dt_socket,suspend=n”

关闭缓存

开发模式下自动关闭模板缓存spring.thymeleaf.cache

测试

集成测试，多个组件配合
@RunWith(SpringJUnit4ClassRunner.class)在单元测试中引入Spring上下文逻辑，高版本可使用简化别名@RunWith(SpringRunner.class)
@ContextConfiguration只引入配置类本身，不会引入额外spring boot相关配置，可以使用@SpringBootTest指定application类

测试Controller端点方法，单纯测不能测出页面行为

  • Mock MVC模拟Servlet容器，SpringBootTest默认模式
    • MockMvcBuilders.standaloneSetup 手动指定Controller,单一测试
    • MockMvcBuilders.webAppContextSetup 根据上下文载入全部，整体测试
  • 嵌入式Servlet容器，真实执行
    • SpringBootTest设置RANDOM_PORT模式，表示要求启动随机端口嵌入环境测试

自定义starter

流程

  1. 原理启动时扫描，寻找包含spring.factories文件的JAR包
  2. 文件指定了自动配置类
  3. 根据配置类条件判断是否注入bean

自定义步骤

  1. 定义pom，引入相关依赖，需要spring-boot-autoconfigure和spring-boot-configuration-processor，或者直接引入总依赖spring-boot-starter
  2. 定义配置类
  3. 定义相关业务类
  4. 定义自动配置类，配置类本身以及内部bean都可以条件控制
  5. src/main/resources/META-INF/spring.factories
    org.springframework.boot.autoconfigure.EnableAutoConfiguration=自动配置类

条件

类控制
@ConditionalOnClass
@ConditionalOnMissingClass

实例控制
@ConditionalOnBean
@ConditionalOnMissingBean

配置项控制
@ConditionalOnProperty

资源控制
@ConditionalOnResource

Web项目控制
@ConditionalOnWebApplication
@ConditionalOnNotWebApplication

Java版本控制
@ConditionalOnJava

配置文件注入

@ConfigurationProperties(prefix=“local”) 本身不会容器，可以定义配置文件和配置类的对应关系
@EnableConfigurationProperties(XxxProperties.class) 注入配置类到容器
