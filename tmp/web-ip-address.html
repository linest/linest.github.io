IP地址

IP地址是网卡MAC地址的通讯地址，具有定位功能
同一个局域网内可以通过MAC来找，网外用IP来找，即不同网的两个设备即使MAC被修改成一样也不影响

IP地址结构

32位，一般划分为两部分：网络号和主机号

私有ip地址段

  • 10段：10.0.0.0~10.255.255.255
  • 172.16-172.31段：172.16.0.0~172.31.255.255
  • 192.168段：192.168.0.0~192.168.255.255

分类

  • A类: 前一段网络号，后三段主机号，二进制0开头，每个网络可容纳16777214
  • B类：前两段网络号，后两段主机号，二进制10开头，每个网络可容纳主机数65534
  • C类：前三段网络号，后一段主机号，二进制110开头，每个网络可容纳主机数254
  • D类：多播地址，二进制1110开头
  • E类：预备保留，二进制1111开头

IPv6

128位，是原来的4倍

  • 地址更多：可以实现每人分配固定ip
  • 路由表小: 分配遵循聚类原则，相同区域前缀相同
  • 安全性：原生IPSec加密传输

CIDR

无类型域间路由(Classless Inter-Domain Routing)
解决原来固定划分ABC类网址，大小不合适，自定义划分位数

  • 主机号全0为子网掩码
  • 主机号全1为广播地址，即该子网络下所有主机都能收到
    IP地址和子网掩码与操作，可以获取网络号

特殊地址

0.0.0.0 未知的源主机，通过DHCP无法获取合法地址
255.255.255.255 全广播

::1 DESKTOP-HKEN0TC

127.0.0.1 回环地址，用于本机通讯，即内核处理后直接返回，不会再网络中出现

查看IP

  • Win: ipconfig
  • Linux: ifconfig或ip addr(源自新旧两套工具)

DHCP

动态主机配置协议(Dynamic Host Configuration Protocol)
机器IP不固定，动态租用获取

  1. 新机器使用0.0.0.0未知身份，发送255.255.255.255广播，包含自身MAC信息
  2. DHCP服务器收到了，根据MAC分配一个IP，依然广播发送，包含子网掩码，网关，IP和租用期限等信息
  3. 新机器可能收到多个，再次广播，通知接受哪个IP
  4. DHCP服务器广播应答
    租期过半时可以发起续租

整个过程中新机器和DHCP服务器互不知情，采取广播方式

ARP

网络层协议
向目标通信，必须知道目标的MAC
MAC作用是网卡收到数据，发现MAC是自己，就进行处理
已知目标IP,需要知道MAC，广播询问，目标应答
收到目标回复，缓存这个MAC一段时间构建ARP映射表，避免短时间内每次都询问

网关

网关和当前机器IP地址要在一个网段，作用是向其他网段转发
路由器，交换机或者是PC都可以作为网关

发送时，已知自身MAC，自身和目标IP
通过自身ip与子网掩码计算网络，判断目标IP是否是同网段

  • 同网段：ARP获取目标MAC，直接发给目标，不通过网关
  • 不同网段：ARP获取网关MAC，发给网关，靠网关转发，网关知道通向其他网段下一跳地址

传输过程数据变化

MAC层数据先目标MAC，然后是来源MAC，因为网卡首先要判断是不是发给自己的
IP层数据先来源IP，然后是目标IP，因为要判断是否一个网段

MAC改变：
每经过一个路由节点，目标MAC会改变，因为发往不同设备

IP改变：
NAT(Network Address Translation)
公网ip很宝贵，不可能每台机器都有公网ip，内部采用内网ip
一般整个内网有部分ip可以访问公网，构成NAT地址池
内网ip请求时都通过NAT路由器，从地址池中挑一个能访问外网的ip，响应结果在反向映射返回给内网ip

端口改变：
由于共享外网地址，如果使用同一端口将建立多对一关系，返回时无法区分来源，因此要使用NAPT技术，ip改变同时端口也要改变，内网ip:port–>外网ip:空闲port

集线器与交换机

两者都可以多机互联

  • 集线器(Hub)
    纯物理层工作，一层设备，收到数据就无脑转发给每个接口，由机器自己判断需不需要，小范围可以，多了比较浪费带宽
  • 交换机(Switch)
    数据链路层工作，二层设备，解析MAC，策略转发
    交换机一开始也不知道目标MAC是哪个接口，但是每次知道来源MAC是哪个接口，慢慢学习出MAC和接口的对应关系，形成转发表

如果链路成环，会发生广播风暴

ICMP

互联网控制报文协议(Internet Control Message Protocol), 网络层协议，工作在IP层上面
提供两种报文，查询报文和异常差错报文

  • ping基于查询报文实现，构建一个ICMP查询报文，发送出去
  • traceroute基于差错报文实现，故意发一个不可能的端口值的UDP，强制目标报错返回ICMP差错报文。并且使用TTL递增，每次多一个路径节点，探测路径上所有的ip
    有些设备禁止ping或者不响应traceroute，不代表不连通

IP

包含TTL字段(Time to live)每次转发后减1，避免报文在网络中的循环转发，浪费网络带宽
