Tomcat

架构

由conf/server.xml提供配置

Server是顶层结构，监听端口用于接受命令
Service包含容器和连接器。一个Service可以包含多个连接器，只能包含一个容器
同一Service下共享连接器，因此监听端口相同。可以配置不同Service，监听另一组端口
连接器配置有监听端口，协议，转发端口等，负责具体连接
容器是分层结构，引擎Engine只有一个，可包含多个Host，每个站点对应一个域名，关联一个目录
站点可包含多个应用Context，每个Context是一个独立应用，关联一个子路径

Context部分直接配在server.xml中，只会在Tomcat启动时加载一次，灵活性不高
可以作为context.xml单独配置，可以放在服务端conf目录下，也可放在应用端META-INF目录下

另外容器还有一个层级Wrapper,对应Servlet。 一般由应用端web.xml配置。 服务端conf目录下也有web.xml作为全局配置。比如处理Jsp的Servlet

类加载

针对问题

  • 容器内不同应用所依赖的类库版本不同，隔离允许每个应用依赖不同版本
  • 容器内不同应用如果依赖相同类库版本，允许共享
  • 容器自身也依赖类库，为了安全不和应用的依赖混合

在常规的类加载体系下，继续扩展

  • Application ClassLoader: 脚本指定，{CATALINA_HOME}/bin
  • Common ClassLoader: 对应common，服务器和应用共享，{CATALINA_HOME}/lib
  • Catalina ClassLoader: 对应server，服务器独占，默认不开启
  • Shared ClassLoader: 对应shared, 应用间共享，默认不开启
  • Webapp ClassLoader：应用类加载WEB-INF/lib WEB-INF/classes，应用独占, class优先lib
  • Jasper ClassLoader：对应单个JSP文件，JSP更新就重新建一个加载器，实现HotSwap

conf/catalina.properties可以配置common.loader，server.loader，shared.loader对应不同的加载器

加载流程

违背双亲委派

  1. 先判断缓存有没有是否已经加载
  2. 尝试让AppClassLoader加载，保障基础库不被覆盖
  3. 尝试WebAppClassLoader自己加载，优先应用间隔离
  4. 最后尝试Common加载，应用共享

线程模型

BIO protocol =" org.apache.coyote.http11.Http11Protocol" Tomcat7及之前默认
NIO protocol =“org.apache.coyote.http11.Http11NioProtocol” Tomcat8开始默认
APR protocol =“org.apache.coyote.http11.Http11AprProtocol” 调用APR库，操作系统级别异步IO，需要额外安装
AIO protocol =“org.apache.coyote.http11.Http11Nio2Protocol” Tomcat8支持

Acceptor线程，从队列中accept建立好的连接
Poller线程维护Selector
Worker线程

acceptCount 新来请求，等待acceptor操作的队列,默认100
acceptorThreadCount acceptor线程数，默认值1，多核下可以每个核一个
maxConnections 最大连接数，超过后acceptor线程阻塞，并不阻止新建连接，而是阻止accept，默认10000
maxThreads 工作线程数，默认200

代码

模式
LifecycleBase 模板方法
ApplicationFilterChain Pipeline-Value 责任链
LifecycleListener 观察者

热加载和热部署

  • autoDeploy: 部署目录中增加新的目录和war,修改web.xml时会重新部署启动
  • reloadable: 替换jar或修改class时，不重启修改生效
    配合Eclipse时,都关闭，在debug环境下修改方法可以不重启

路径

  • url路径: Context中path指定
  • 部署目录: Host中appBase指定，默认是webapps
  • 资源路径: Context中docBase指定，可绝对路径或相对路径，相对于根目录
    Eclipse下项目部署在
    workspace\.metadata\.plugins\org.eclipse.wst.server.core\tmp0\wtpwebapps

配置数据库资源

server.xml,数据库driver放在tomcat的lib目录下

生产环境JVM参数

  • -server
  • -Xms-Xmx：设成一样，预先分配避免变化
  • -Xss: 线程栈，默认为1M，设512k/128k
  • -XX:PermSize和-XX:MaxPermSize=512m
  • -XX:+UseBiasedLocking: 偏向锁优化,每个请求一个线程
  • -XX:LargePageSizeInBytes: 分页页面 128M
  • -XX:+UseFastAccessorMethods: get/set方法本地化
  • -XX:MaxTenuringThreshold: 晋升老年代所需次数 31

Connector参数

  • 编码控制：URIEncoding=“UTF-8”
  • 线程控制：maxThreads=“300” minSpareThreads=“25” maxSpareThreads=“75”，一般来说线程数200~500，需根据配置压测
  • 关闭查找：enableLookups=“false”
  • 超时：connectionTimeout=“20000”
  • 延长上传超时时间:disableUploadTimeout=“true”
  • 等待队列：acceptCount=“300”
  • 压缩：compression=“on” compressionMinSize=“2048” compressableMimeType=“text/html,text/xml,text/javascript,text/css,text/plain”
