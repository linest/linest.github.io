spring-bean

spring-beans

BeanPostProcessor

Bean功能扩展的核心接口，Bean初始化过程中嵌入自定义逻辑
ApplicationContext能自动感知并注册已有的BeanPostProcessor,比较原始的BeanFactory需要addBeanPostProcessor手动注册
BeanPostProcessor注册顺序并不代表执行顺序，需要实现Ordered接口或者@Order注解

InstantiationAwareBeanPostProcessor也是PostProcessor，增加了实例化过程的相关方法
自定义实现时通常使用InstantiationAwareBeanPostProcessorAdapter适配器类

@PostConstruct和@PreDestroy对应InitDestroyAnnotationBeanPostProcessor
实现PriorityOrdered接口，相比普通的BeanPostProcessor更加优先

BeanFactoryPostProcessor

容器也有后处理器，只在容器初始化时执行一次
BeanFactoryPostProcessor目标是修改BeanDefinition，调用时BeanDefinition加载完成，但是尚未实例化

Bean的声明周期

创建过程

实例化(Instantiation)

  1. 先执行InstantiationAwareBeanPostProcessor的postProcessBeforeInstantiation方法
  2. 执行构造
  3. 执行InstantiationAwareBeanPostProcessor的postProcessAfterInstantiation方法
  4. 执行InstantiationAwareBeanPostProcessor的postProcessProperties方法

注入属性

  1. 注入属性
  2. 调用BeanNameAware的setBeanName
  3. 调用BeanFactoryAware的setBeanFactory

初始化(Initialization)

  1. 执行BeanPostProcessor的postProcessBeforeInitailization
  2. 调用InitializingBean的afterPropertiesSet
  3. 调用initMethod
  4. 执行BeanPostProcessor的postProcessAfterInitailization
  5. 如果实现SmartInitializingSingleton执行afterSingletonsInstantiated

至此创建完成，如果是单例，放入容器缓存继续管理生命周期，如果是原型交给调用者管理

销毁过程

  1. 执行DestructionAwareBeanPostProcessor
  2. 调用DisposibleBean的destroy方法
  3. 调用destroy-method

个体初始化嵌入逻辑

针对个体，有三套方案在构造和销毁时插入逻辑，三套方案BeanPostProcessor先于接口先于方法

  • @PostConstruct和@PreDestroy来自于javax.annotation,可以标注多个方法
  • InitializingBean和DisposableBean接口来自spring-beans,有一定侵入性
  • initMethod和destroyMethod来自spring-context的@Bean
  1. 构造函数
  2. @Autowire注入
  3. @PostConstruct
  4. InitializingBean接口afterPropertiesSet方法
  5. @Bean(initMethod=“init”)
  6. @PreDestroy
  7. DisposableBean接口destroy方法
  8. @Bean(destroyMethod=“destroy”)

所有这些机制只有Spring托管下才会维护，自己new出来的无效

Scope

默认单例
好处是像XxxService一般是无状态的，没必要创建多个，并且Spring创建对象涉及代理比较费时，共享后减少了创建同时减少回收
@Scope(value = ConfigurableBeanFactory.SCOPE_SINGLETON)
@Scope(value = ConfigurableBeanFactory.SCOPE_PROTOTYPE)

Web相关，启动后并没有请求，先创建代理来注入，使用时再新建
@Scope(value = WebApplicationContext.SCOPE_REQUEST, proxyMode = ScopedProxyMode.TARGET_CLASS)
@Scope(value = WebApplicationContext.SCOPE_SESSION, proxyMode = ScopedProxyMode.TARGET_CLASS)
@Scope(value = WebApplicationContext.SCOPE_APPLICATION, proxyMode = ScopedProxyMode.TARGET_CLASS)
@Scope(scopeName = “websocket”, proxyMode = ScopedProxyMode.TARGET_CLASS)
