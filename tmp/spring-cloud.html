spring-cloud

Spring Cloud Config

分布式配置

注册中心，服务发现，解决服务去哪调用的问题

  • Eureka
  • Consul

Feign

无需手动操控HttpClient发起请求，直接定义接口，通过动态代理模式发起请求，简化调用复杂性

Ribbon

多实例下负载均衡，选择一台机器发送请求
还可控制超时和重试

调用隔离熔断

熔断器，发现服务不可用，不进行无用请求，可以走降级备用方案，避免单一服务不可用造成级联崩溃

隔离类型

  • 线程池隔离：
    每个业务分配线程池，单一服务不可用只会充满关联线程池，其他服务线程池不受影响
    请求线程和执行线程不一样，请求线程不会阻塞，缺点是线程池开销(排队，调度，上下文切换)，适合网络调用
  • 信号量隔离：
    请求线程和执行线程是同一线程，信号量控制并发，拿不到信号就阻塞等待或者降级
    适合不涉及网路请求的内部调用，内部调用一般不会阻塞延时

线程数量和超时时间，根据QPS和处理时间来定
线程太少或者超时时间过长会导致无线程可用

A服务每秒请求B服务30次，需要200ms收到结果，理论上1s内单线程可以发5次请求，需要6个线程，冗余成10个线程。 10个线程，每秒单线程发3次即可，即时间可以容忍300ms，即超时时间300ms, 不管成功与否都返回，保证有线程可用。

框架

  • Hystrix
    • 线程/信号量隔离
    • 失败比率
  • Sentinel
    • 信号量隔离
    • 响应时间/失败比率
    • 支持限流，慢启动，匀速
    • 完善的监控

Zuul

服务网关路由
所有请求统一入口，网关根据请求特征(比如来自移动端，PC端)导向不同后端服务器
