Java Stream

来一个处理一个~

Stream顾名思义是一种流结构，是Java8亮点之一。由于配备了lambda功能，更加简洁，是for循环处理的有利竞争者。

集合类最常见的操作就是遍历，通过对每个元素进行处理，达到特定目的。想象数据库表是一个List,那么Stream操作可以实现与SQL类似的语义，筛选出符合条件的结果。

Stream的好处

  • 强语义，代码简洁，更易理解
  • 松耦合，各个环节相互独立
  • 不可变，产出新数据而不是修改原数据
  • 分布式，提供并行流，MapReduce语义

坏处

  • 低性能，不如for循环，以后的版本可能会改善
  • 难调试，单行代码不容易跟踪

Stream操作

Stream配备的处理方法在java.util.stream.Stream接口中定义。

假定有一个人员列表充当数据库List DATA，用stream模拟各种操作

利用map方法将每个对象转成另一个对象

计数

利用count方法取得流数量

Max/Min

利用max,min方法取得极值

Top

顺序Top 1，利用findFirst方法取得第一个值

逆序Top k，利用limit方法限制数量

Stream收集器

除了返回数值和单体值，Stream处理的结果还是流，需要收集器转成其他结构，就像水装进瓶子一样。
java.util.stream.Collectors预定义了一组收集器

收集成list/set

上文已多次使用了Collectors.toList()，无参数直接使用
类似还有Collectors.toSet()

收集成map

分别传入key和value的生成方法

收集成统计信息

一次性获得多种统计信息

分组

Collectors.groupingBy提供一个函数计算出key，把相同key的数据收集成list，返回一个map，即每个key对应一个value列表。

groupingBy默认使用HashMap,值是List。可以更换默认类型。

Stream有map方法，Collectors同样提供了mapping方法可以对分组值进行映射变换

并行流

数据分散到可以多线程并行计算，多核场景下性能可以提升

如果不指定并行流，全部由主线程执行，并且不会用到reduce的第三参数
指定并行后，数据分散给不同线程计算，第三参数combiner用于合成不同线程的结果
从线程名可以看出，多线程底层是基于ForkJoinPool实现
另外两种方式的计算结果不同，这是因为指定的初始值虽然都是1，但是多线程下每个线程都进行了初始，相当于多了内容
