Java HashMap 与 HashSet

  • 源码
Java

基于Hash的集合

代码版本jdk1.8.0_101

HashMap顾名思义就是利用哈希机制构建的Map
应该是Java最常用的Map结构

底层存储结构

HashMap的基础结构是Node数组，Node是Entry的子类，一个node表示一条数据
数组容量必须是2的幂次，默认初始容量16， 最大容量2的30次幂

由数据映射到底层

每当新加入一条数据，需要决定数据的存放位置

  1. 首先，对key求哈希值，将hash作为存放位置的依据
  1. hash的范围是整个int空间，还需要将hash映射到当前的数组大小
    由于整数n的大小始终为2的幂次，那么n-1的二进制表示为末尾全1的形式，相当于一个掩码，把hash值截断，相当于取模运算，比%效率更高
  1. 截断的hash值损失了高位信息，因此采用了一种弥补方式
    计算哈希时并没有直接采用key的hashCode,而是与高16位进行异或的结果作为hash值

哈希冲突解决

Hash表冲突有两大解决方案

  • Open addressing:一个位置由一条数据独占，如果发现目标位置被占用，就按规则选其他位置
  • Separate chaining:一个位置被多个数据共享，这些数据链接起来存储
    Java的HashMap冲突解决策略采用后者。
    • Java7中，所有冲突都用链表解决。哈希全部冲突的极端情况下，退化成链表，查询效率变成O(n)。
    • Java8中，才用两种方案。如果同一位置的数据比较少，线性查找尚可接受，那么直接链表存储，如果比较多(默认是超过8)则转化成一颗红黑树来优化查询效率。这种转化是双向的，如果删除数据，那么树可能重新退化成链表。 因为采用了双结构，因而实现上有两种形式的Node,一种为普通的链表Node,除了包含hash,key,value这三个基础元素，还包括next引用。另一种为树节点TreeNode,它在Node基础上增加了parent,left,right,prev引用以及红黑树用到的颜色信息。

插入时，当发现位置被占用

  1. 首先if判断是不是同一个key，如果不是那就要新增一个数据
  2. 然后if判断是不是树结构，如果是则插入新的树节点
  3. 最后说明是链表结构。新增链表节点的同时考虑要不要转化成树。默认的TREEIFY_THRESHOLD阈值是8。

检索方式

  1. 由key的hash值定位到底层Node
  2. 判断当前Node是否命中
  3. 不是的话在Node关联的其他Node中继续查找，根据Node类型进行链表查找或树查找

判断命中有两层条件，首要条件是hash值相等，进而判断同一key或key相等
这就是为什么自定义对象用作key时要重写hashcode和equals方法

扩容

初始时HashMap容量为0，并不进行存储分配,因此只要不插入数据就没有数据存储空间
put和putAll操作会导致容量扩充,扩容的标准是判断是否超过threshold

用户设置的初始容量值可能不是2的幂次，会根据容量计算出一个threshold值，他是可以满足需求的最小的2的幂次值。2的幂次有利于扩容、取模等操作。

扩容的基本标准是倍增，阈值threshold的基本标准是容量和loadFactor的乘积。默认的loadFactor为0.75。可见并不是满了才扩容，而是快满了就会扩容。0.75是空间和查询时间的tradeoff。如果很满，说明大概率哈希冲突，可能同一位置数据过多，从而降低查询效率。

  • Java6中扩容阈值始终采用因子乘积
  • Java7中第一次扩容阈值采用初始容量，之后采用因子乘积

由于底层容量改变，计算存储位置时会发生变化，因此扩容之后需要对数据进行重新hash。
Java8之前扩容采用全部重新hash的方式，如果HashMap被用到高并发环境下，同时扩容，可能出现链表环，从而get时死循环。
Java8优化了扩容，无需全体重新计算hash.由于是倍增方式扩容，相当于掩码增加了一位，相当于在高位多了0或1。 因此只有两种情况，要么保持不动，要么移动到新增容量的同一位置。
Java8的HashMap采用的策略是保持相对顺序，扩容之前和之后Node链表内数据顺序保持不变。而Java7扩容后同一位置数据顺序可能更改。
建立两个链表loHead和hiHead分别表示低位和高位，将原链表按照高位值不同划分到两个子链表，loHead保持原链表位置，hiHead挂到新增区域的对应位置

遍历

Map可以按照三种方式遍历：keySet,values,entrySet
这三个对象都是无状态的，不存储任何信息，遍历直接遍历底层Node数组
值得注意的是modCount变量，它记录了HashMap变动数，每次操作都会增加，可以作为fail-fast机制的标志变量
一旦在遍历前后modCount发生变化，直接抛异常。

使用技巧

建HashMap时传入初始容量参数，因为扩容代价很大
如果没有目标容量，那么设为16，如果有设为目标容量/负载因子+1

HashSet

有了HashMap之后，HashSet的实现变得极其简单
HashSet和HashMap的区别是没有Key-Value结构

实际上内部就是个HashMap,Map所有的value都存成一个用于占位的Object
